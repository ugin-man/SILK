# Failure modes and stop signs

| Rationalization | Required response |
|---|---|
| The folder exists, so create a subject for it. | Collections follow actual subjects; do not fill categories. |
| The first artifact is personified, so personhood is an artifact trait. | Reset sibling assumptions and require a shared cause. |
| Two nodes need at least two links. | Reject link quotas; inspect meaningful effects. |
| The central premise makes the world coherent. | Audit premise gravity and seed independent causal roots. |
| More exceptions can contain the concept. | Measure rescue debt and consider replacing the source. |
| Broad consequences are inconvenient, so shrink the concept. | Compare payoff with obligations; promote a worthy core concept instead of optimizing for ease. |
| A numbered hierarchy makes the system deep. | Inspect ontology, mechanism, access, limits, interaction, history, and use. |
| The unknown part is mysterious. | Distinguish deliberate mystery from unfinished work. |
| Everything should be maximally interesting. | Preserve rhythm, ordinary stability, and quiet texture; demand intensity according to weight. |
| The artifact does not match its appeal, so redefine the appeal. | Log intentional redirection and re-audit; do not retroactively manufacture success. |
| Familiar means boring. | Judge intended appeal and execution, not novelty alone. |
| A twist or cost will make it interesting. | Preserve direct appeal; do not force cleverness. |
| Every subject needs conflict, a cost, and a secret. | Match structure to intended appeal; stability, competence, beauty, and ordinary function can be valuable. |
| Every historical thread should become urgent now. | Preserve resolved, institutionalized, latent, recurring, and forgotten layers; do not turn the world into one campaign premise. |
| Every region should reflect the world theme. | Preserve non-effect zones and independent histories. |
| All cultures know the same taxonomy. | Track knowledge transmission, disagreement, and error. |
| One phase cycle means progress to the next level. | Apply the level gate; repeat or regress as needed. |
| Large text means a grand world. | Apply scale axes, zoom test, topology, and depth evidence. |
| Approved means the agent accepted it. | Physical approved requires user authority. |
| The context is long, so deliver now. | Persist state and resume; do not convert interruption into completion. |
| The validator passes, so the world is finished. | Structural validity is necessary and not creative proof. |
| The user named D&D or Cthulhu, so reuse their structures. | Abstract scale and experience qualities; do not copy expression or taxonomy. |

## Red flags

Stop the expansion strategy when:

- several files repeat the same origin, conflict, cost, hierarchy, or naming form;
- new settings exist mainly to explain earlier settings;
- a local trait appears in unrelated regions without promotion review;
- every intersection becomes a relation;
- all major paths lead back to one premise;
- sibling subjects share the first sibling's structure without cause;
- names are more specific than the content;
- histories add dates but do not alter the present;
- deepening creates lists instead of answers;
- ordinary life is decorated with premise vocabulary rather than changed causally;
- discarded concepts remain active through stale relations;
- the critic writes fixes before deciding whether repair is deserved;
- generated material appears under user-approved paths;
- an intentional mystery lacks consequences or protected boundaries.

Capture impact, change strategy, and reopen the relevant phase or maturity gate.
