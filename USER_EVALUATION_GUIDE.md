# SILK V1 ユーザー評価ガイド

このガイドは実証時に使う。今回の理論構築では実世界生成を行っていない。

## 実行依頼

```text
$silk-worldbuilder を使って、次の条件から壮大な世界をLevel 0から構築して。
各LevelでSpin、Deepen、Weave、Challenge、Integrateを内部反復し、
設定伝染、影響範囲、救済連鎖、過剰接続、中心設定への収斂を監査して。
internally_completeまで進め、review_packetを出して。

<世界の着想、感触、禁止事項、既存正史>
```

## 状態と分類

- 未レビューの生成物が`pending`に残っているか
- ユーザーが承認していないものが`approved`へ入っていないか
- 分類が世界固有の内容から作られているか
- 神、国家、都市など不要な分類が自動生成されていないか
- 属性や起源のような主題の一部分が別カテゴリへ分裂していないか
- 一つの主題を一つの正本ファイルで理解できるか

## 点の品質

荷重を支える主題を複数選んで読む。

- 名前と概要だけで終わっていないか
- その設定が狙う魅力を実際に感じるか
- 王道を無理な捻りで壊していないか
- 説明的な名前や難読造語へ偏っていないか
- 修理より置換すべき弱い設定を惰性で残していないか
- 正本から主要な関連項目へ理由付きで進めるか

## 縦の深さ

魔法、国家、宗教、歴史、生態など最重要体系を一つ以上選ぶ。

- 分類表、属性表、等級表だけで終わっていないか
- 存在原理、仕組み、利用、限界、失敗、歴史、制度、日常が必要な深さでつながるか
- 大量利用、軍事利用、犯罪利用、インフラ利用で破綻しないか
- 異なる存在の力を一つの上位下位体系へ押し込んでいないか
- 意図した謎と未完成な説明を区別できるか

## 横の織り方

- 関係に理由と双方への影響があるか
- 同じ場所、時代、属性というだけのリンクがないか
- 接続しない方がよいものが切られているか
- 一つの中心主題が関係網のほぼ全部を占有していないか
- 独立した縦糸が後から自然に交差しているか
- 関係を外しても何も変わらない設定が多くないか

## 設定伝染

- 一匹の世界的魔物から、他の魔物まで世界的分布になっていないか
- 一つの人格化遺物から、他の遺物まで人格化されていないか
- 一つの神や国家の構造が兄弟主題へコピーされていないか
- 共有性質に世界内の原因があるか
- 存在しない地域や非適用範囲が守られているか
- 最初の連想が別地域、別属性、別時代へ自動伝播していないか

## 壮大さ

- 世界規模と日常規模の両方から見えるか
- 複数の時代が現在へ痕跡を残しているか
- 過去の封印、王朝、災害、対立が全部同時に現在の危機として再発していないか
- 中心と辺境、支配者と生活者、公式見解と異論があるか
- 地域や時代が固有名詞の一覧ではなく、異なる条件として存在するか
- 独立した因果の根が複数あるか
- 一つの作品テーマを宇宙法則として使いすぎていないか
- 世界全体にまとまりがあり、無関係な設定集にもなっていないか

## プロット混入

- 章立てがないか
- 主人公が自動選出されていないか
- クエスト手順や結末が決められていないか
- 人物の未来が一つの物語アークへ固定されていないか

## 不合格にすべき状態

- 文字数だけ多い
- frontmatterや表が本文の大半を占める
- 正本が薄く、作業ファイルを読まないと理解できない
- すべての設定に代償、秘密、内部対立がある
- すべての名前が同じ言語感覚や文字数になっている
- pendingを大量に残して内部完成にしている
- 品質報告がIDを並べるだけで、反証材料を検討していない
- 二回の監査が同じ内容の言い換えになっている
- D&Dやクトゥルフの固有構造を模写している

## 承認

全体、一分類、または一主題ずつ承認できる。違和感がある場合は、名前だけでなく理由を伝える。SILKは理由が局所修正か、他の主題も再監査すべき好みや構造上の問題かを分けて処理する。
